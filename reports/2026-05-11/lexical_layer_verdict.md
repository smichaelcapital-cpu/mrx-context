## What happened

At 7 pairs, lexical was NOT_READY and the 7-pair report noted: "at 195 pairs, lexical should cross HIGH." That prediction did not hold this run — not because the patterns don't exist, but because the measurement tool broke on the new input format.

The v2 rough extraction (used for the first time at full 195-pair scale) concatenates all text-bearing lines from the SGCAT32 binary sgngl file. This includes steno dictionary entries stored in the file. These entries appear in the concatenated rough as doubled-word artifacts:

```
Example rough segment: "pastorpastor" "panepain" "laplaceleplace" "catherinekathryn"
```

When the SequenceMatcher alignment tool tries to find where rough word "pastorpastor" aligns to final word "pastor", it records "pastorpastor" → "pastor" as a substitution event. These artifact substitutions overwhelm the real lexical signal (there→their, your→you're, etc.).

Specific known patterns from the 7-pair run:
- `there → their`: 3 events (was 9 at 7 pairs) — below noise floor
- `your → you're`: 4 events (was 6 at 7 pairs) — below noise floor

The steno artifact events number ~25 per pair vs ~1 real lexical event per pair — a 25:1 noise ratio.

## What would change the verdict

### Option A (recommended): Steno-code-filtered rough extraction

Identify and strip steno dictionary entry lines from the sgngl before alignment. Steno entries follow recognizable patterns (doubled words, steno code characters). A filter that removes these lines before the rough text is concatenated would reduce noise substantially.

Estimate: if this filter removes 80%+ of the artifact substitutions, the real lexical signal (there→their appearing in ~40%+ of pairs) should cross the SUGGEST threshold. Getting to HIGH requires that signal to appear in 80%+ of pairs — possible at 186-pair scale if the filter is effective.

### Option B: Use only transcript-bearing lines

The sgngl structure stores actual transcript text in specific line positions. If the extraction can be made to isolate ONLY those lines (not dict entries), alignment quality improves dramatically. Requires deeper sgngl format analysis.

### Option C: Fallback to Q./A. final-only analysis

Without rough comparison, measure lexical patterns ONLY in the final text: word frequency, unusual word choices, spelling preferences. This doesn't measure rough→final transformation habits but does produce a final-text fingerprint. Lower evidential value for MB's editing habits, but executable now.
