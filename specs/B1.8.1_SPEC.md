## Goal

Build a minimal, human-readable side-by-side viewer that takes a depo stem and produces two output files:
1. `review/<stem>_sxs.txt` — side-by-side, ours left column / MB oracle right column, page by page
2. `review/<stem>_diff.txt` — unified diff showing only lines that differ

Scott reads these files to see what's actually rendering vs what MB ships. No pass/fail gate yet — just render and write.

## Inputs

CLI usage:
python harness/fm_review.py <depo_stem>

Where `<depo_stem>` is one of: halprin, 060122williams, 082222butler, 101322blanks, 032025olsen, 0525black_bp.

## File paths the script must resolve

**Rendered output:**
io/analysis/<stem>/_front_matter_out/<stem>.front_matter.txt
If missing → exit 1 with message: "Rendered file not found. Run front matter pipeline first."

**Oracle:**
../mrx-context/oracle/finals/<stem>/
Look in that folder for a file matching `*-FINAL.txt`. If the folder or file is missing → exit 1 with a clear message naming the path it tried.

Special case: halprin oracle filename pattern is `040226yellowrock-FINAL.txt`. Use a glob on `*-FINAL.txt` to find the file in the folder — do not hardcode filenames.

## Output paths

Create `review/` folder in the repo root if it doesn't exist (gitignore it via `.gitignore` update — review files are local working output, not commits).

Write two files:
- `review/<stem>_sxs.txt`
- `review/<stem>_diff.txt`

## Side-by-side format

- Split both files on `\x0c` (form-feed = page break)
- For each page, line up ours and oracle by line index
- Each output line: `<OURS column, fixed width 80 chars, space-padded or truncated> | <ORACLE>`
- Page separator: a line of `=== PAGE N ===` between pages
- If one side has more lines than the other on a page, show empty string on the shorter side
- Width 80 is the working assumption — adjust if oracle lines exceed 80, but keep the format readable

Example output:
=== PAGE 1 ===
1                                                 |                             1
|
1              UNITED STATES DISTRICT COURT                              |      1              UNITED STATES DISTRICT COURT
2              EASTERN DISTRICT OF LOUISIANA                             |      2              EASTERN DISTRICT OF LOUISIANA
...
=== PAGE 2 ===
...

## Unified diff format

Use Python's `difflib.unified_diff` with context lines = 3. Header lines should be:
--- ours: io/analysis/<stem>/_front_matter_out/<stem>.front_matter.txt
+++ oracle: <full path to oracle file>

If the two files are byte-identical, write a single line: `IDENTICAL — no diffs.`

## Exit codes

- 0 — both review files written successfully (regardless of whether content matches)
- 1 — rendered file or oracle file missing (with clear stderr message naming the missing path)
- 2 — unexpected error (log the traceback)

The script does NOT gate on diff content. Identical and different both exit 0.
