## Change 1 — Force pending=False before trailing blank in each firm block

**Function:** `_paginate_blocks` in `src/stage5/appearances_renderer.py`

**Approach:** Identify the trailing blank of each firm_block and ensure it always produces a 2-row blank slot.

**Recommended implementation:**

In the block-emission loop (around lines 264-265), detect when emitting the trailing blank of a firm block. Before that emit, set `pending = False`. This forces `emit_item`'s blank-handling branch to take the `cur.extend([_BLANK_ROW, _BLANK_ROW])` path.

Alternatively: bypass `emit_item` for the trailing blank entirely. If the previous row left `pending=True`, close it (leave the sub-row as-is) and add 2 fresh blank rows.

Pick whichever is cleaner in the existing code structure. Both approaches are equivalent in output.

**Detection of "trailing blank of a firm block":**
- Each firm_block ends with a `_ll("", INDENT_ROLE)` per line 348 of `build_appearances_pages_v2`
- That's the last tuple in each `firm_block` list — index `[-1]`
- Inside `_paginate_blocks`, when iterating items within a block, the last item that is `(0, "")` (or whatever the constant produces) is the one to force-close before

## Change 2 — Update `_block_row_cost` to match

**Function:** `_block_row_cost` (lines 166-188 per Sonnet's recon)

The cost function simulates pagination cost using the same pending-state logic. After Change 1, the trailing blank of each firm block always costs 2 rows. The cost function must reflect this or KEEP-TOGETHER pagination will mis-estimate.

**Required:** In `_block_row_cost`, when scoring the trailing blank of a firm block, count it as +2 rows regardless of pending state at that point in the simulation.

**Implementation:** Apply the same "force pending=False before trailing blank" logic in the cost simulator. The two functions must stay in lock-step.
