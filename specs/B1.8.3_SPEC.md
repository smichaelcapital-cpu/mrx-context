## Goal

Make the side-by-side files readable in any standard editor without horizontal scrolling. Today _SXS_WIDTH=80 means each line is 80 + 3 + 80 = 163 chars wide. Most editors don't fit that.

## What changes

### 1) Reduce column width

In harness/fm_review.py, change the column width constant from 80 to **55**.

Reasoning: Standard transcript page width is 65 chars. Most content fits in 55. Lines that exceed 55 get truncated with `...` suffix (per existing truncation logic — confirm and preserve). Truncation is acceptable for visual scan; full text is in the _diff.txt files anyway.

Result: each line = 55 + 3 + 55 = 113 chars. Fits in any standard editor window.

### 2) Truncation safety check

If the current code truncates with `...` already, leave it. If it truncates without a visible marker, add `...` to make truncation obvious so Scott doesn't think content is missing.

Verify by spot-checking that no content silently disappears.

### 3) Delete stale B1.8.1 review files

The review/ folder still has the old filename pattern from B1.8.1:
  review/<stem>_sxs.txt
  review/<stem>_diff.txt

B1.8.2 introduced new filenames:
  review/<stem>_front_sxs.txt
  review/<stem>_front_diff.txt
  review/<stem>_back_sxs.txt (when back matter exists)
  review/<stem>_back_diff.txt (when back matter exists)

Delete the old _sxs.txt and _diff.txt files (without the _front or _back prefix) for all 6 depos to avoid confusion. Do this as part of the verification run — easier to delete them all in one shot than reason about which is which.

### 4) Re-run harness on all 6 depos

After the width change, run fm_review.py for all 6 depos to regenerate the review/ files at the new width.
