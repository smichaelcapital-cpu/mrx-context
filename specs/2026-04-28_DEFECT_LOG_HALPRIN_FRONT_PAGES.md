## Methodology

**Read / Decide / Write framing:**
- **Raw RTF** = fixed input (what the steno machine produced). We cannot change it.
- **MB FINAL** = fixed truth (what a trained court reporter produced). We cannot change it.
- **Pipeline** = the only thing we control.
- Every defect must trace to one of: we **read** the RTF wrong (Stage 1) / we **decided** wrong (Stage 3 LLM) / we **wrote** wrong (Stage 5 formatter).

**Three-way diff verdicts:**
- A = RTF lacks it, MB inserts during polish → not a pipeline bug
- B = RTF has it, pipeline strips it → confirmed pipeline bug
- C = RTF has it, pipeline preserves it → no defect, visual review error
- D = RTF has it in different form, pipeline doesn't translate → narrow translation bug
