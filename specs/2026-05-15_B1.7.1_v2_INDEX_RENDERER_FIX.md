## Purpose

Fix 6 bugs in the index renderer surfaced by Sonnet #2's deep recon, plus 1 latent positional-trap fix, plus the minimal wire-in to call `build_index_pages()` from `assemble_final.py` instead of hardcoded Halprin page strings. This unblocks front matter rendering for all 6 MB depos.

Wire-in scope is intentionally narrow: **index-only**. Full orchestrator wire-in (cover, stipulation, back matter) is deferred to a future tile (A12 / B1.8). Reason: the new front_matter/cover.py and front_matter/stipulation.py renderers are not yet proven byte-identical to the existing stage5/ renderers for Halprin. Verifying that is its own work. The index renderer is the only piece that was hardcoded Halprin-only, so swapping just the index unblocks multi-depo rendering with the smallest possible blast radius.

## Bug Fixes — Detail

### B1 — Hardcoded count guard
**File:** src/stage5/front_matter/index.py
**Line:** 193
**Current:** `if len(nav) < 6: raise NotImplementedError`
**Fix:** Remove the count guard. Replace with label-based lookup throughout the function. Resolved jointly with B2 and B3.

### B2 — Unconditional positional access
**File:** src/stage5/front_matter/index.py
**Line:** 221
**Current:** `wit = nav[5]`
**Fix:** Use label-based lookup. Resolved jointly with B1 and B3.

### B3 — Hardcoded nav indices for Exam / Rpt Cert / Wit Cert
**File:** src/stage5/front_matter/index.py
**Lines:** 204-221
**Current:** Positional access to nav[3], nav[4], nav[5]
**Fix:** Use label-based lookup. Add a helper `_nav_find(nav, label_substring)` that:
- Lowercases both sides
- Substring-matches (handles "Witness's Certificate" vs "Witness Certificate")
- Returns the entry or raises `KeyError` with a clear message if not found

Label match keys:
- Examination → substring "examination" (matches both "Examination" and "Re Examination" — handle both explicitly by finding "examination" entries in order)
- Reporter's Certificate → substring "reporter"
- Witness Certificate → substring "witness"

### B4 — Only first examiner rendered
**File:** src/stage5/front_matter/index.py
**Line:** 210
**Current:** `exam["sub_entries"][0]`
**Fix:** Loop over all `sub_entries` for the Examination entry AND any Re Examination entry. Williams has 2 examiners (Mr. Leefe, Mr. Gileson) and Re Examination. black_bp has 2 examiners (Mr. McNeal, Mr. Falcon) and Re Examination. Both currently lose the second examiner silently.

### B5 — Null page renders as "None"
**File:** src/stage5/front_matter/index.py
**Lines:** 83, 86, 92, 95
**Current:** `str(page)` called unconditionally
**Fix:** Guard before stringifying. If page is None, render the page field as blank string (`""`). Decision: blank, not literal "NOT MARKED" — the description_lines already carries "NOT MARKED" text per black_bp exhibit 11.

### B6 — witness_cert.json loaded before guard check
**File:** src/stage5/front_matter/orchestrator.py
**Line:** 122
**Current:** `wc_data = _load(depo_stem, "witness_cert.json")` runs before `read_and_sign` check at line 127
**Fix:** Move the load inside the `if read_and_sign:` block at line 127. Butler, Olsen, black_bp, Blanks have no witness_cert.json — current code crashes before reaching the read_and_sign guard.

### NEW-7 — Positional trap at line 204 (`range(3)` loop)
**File:** src/stage5/front_matter/index.py
**Line:** 204
**Current:** `for i in range(3): entry = nav[i]` — accesses nav[0], nav[1], nav[2] positionally for Caption / Appearances / Stipulation
**Risk:** All 6 current MB depos have Caption / Appearances / Stipulation as nav[0-2]. If a future depo doesn't, output is silently wrong.
**Fix:** Use the same `_nav_find()` helper. Look up "caption", "appearances", "stipulation" by label. If any label is not found, raise `KeyError` with a clear message — loud failure, not silent miscoding.

### not_present attorneys — NO FIX NEEDED
Sonnet #1's recon confirmed `src/stage5/front_matter/appearances_renderer.py` line 52 already handles `kind: "not_present"` correctly:
if appearance["kind"] == "not_present":
lines.append(_ll("NOT PRESENT", INDENT_FIRM))
Butler's 2 not-present firms will render correctly. Document in commit message. No code change.

## Wire-In — Index Only

### Current state (assemble_final.py)
- Index pages rendered from hardcoded `_PAGE_2`, `_PAGE_3`, `_PAGE_4` string literals in `appearances_page.py` (Halprin-specific text).

### Wire-in change
In `assemble_final.py`:
1. Import `build_index_pages` from `src.stage5.front_matter.index`.

3. The depo_stem is already known to `assemble_final()` via case_info. Use it to construct the load path.

These become a separate spec: tentatively A12 / B1.8.
