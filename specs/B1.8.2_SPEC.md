## Goal

Make the harness compare like-for-like. Today it diffs our N-page front matter against the entire 300-page oracle FINAL.txt. Result: enormous false-positive diff counts dominated by body text. After this ticket, the harness slices the oracle so we only compare:
- Our front matter pages vs oracle pages 1 through N_front
- Our back matter pages vs oracle last N_back pages (when back matter exists)

## What changes

### 1) Determine N_front and N_back from JSON config

For a given depo stem, read these two files to determine page counts:

**Front matter pages (N_front):**
- Count pages in our rendered output between the start of the file and the page break that separates front matter from back matter
- Simplest implementation: read our combined file `io/analysis/<stem>/_front_matter_out/<stem>.front_matter.txt`, split on `\x0c`, count total pages = N_total
- If back matter exists, compute N_back, then N_front = N_total - N_back

**Back matter detection:**
Read `src/profiles/mb/data/front_matter/<stem>/stipulation.json` and look for the field that indicates witness signature. The exact field name is `witness_reserves_signature` (or equivalent — confirm from the stipulation.json file format you saw during B1.8.1 inventory).

If the field is `true` AND `src/profiles/mb/data/front_matter/<stem>/witness_cert.json` exists → back matter is present.

**Back matter page count (N_back):**
For depos with back matter, you need to know how many pages it occupies. Two ways to get this — pick whichever is simpler:
- (a) Render and count the back matter pages in isolation (call build_back_matter directly, count form feeds)
- (b) Hard-code from observation: today both Halprin and Williams have 5 back matter pages. Document the assumption in code and add a TODO to compute dynamically if it ever varies.

Recommendation: (a) if straightforward, otherwise (b) with the TODO.

### 2) Slice the oracle

Read oracle file (path resolution unchanged from B1.8.1 — glob `*-FINAL.txt` in `mrx-context/oracle/finals/<stem>/`).

Split oracle on `\x0c` → list of all oracle pages = oracle_pages.

Build two slices:
- `oracle_front = oracle_pages[0:N_front]`
- `oracle_back = oracle_pages[-N_back:] if N_back > 0 else []`

### 3) Slice our rendered output

Similar — split on `\x0c`:
- `ours_front = ours_pages[0:N_front]`
- `ours_back = ours_pages[N_front:] if N_back > 0 else []`

### 4) Output files

Replace the two output files from B1.8.1 with four (or three when no back matter):

**Always written:**
- `review/<stem>_front_sxs.txt` — side-by-side front matter, ours vs oracle slice
- `review/<stem>_front_diff.txt` — unified diff of front matter only

**Written only when back matter exists:**
- `review/<stem>_back_sxs.txt` — side-by-side back matter
- `review/<stem>_back_diff.txt` — unified diff of back matter

**Header line in each diff file** should now show what's being compared:
--- ours: io/analysis/<stem>/_front_matter_out/<stem>.front_matter.txt [pages 1-N_front]
+++ oracle: <full oracle path> [pages 1-N_front]

For back diff:
--- ours: io/analysis/<stem>/_front_matter_out/<stem>.front_matter.txt [pages N_front+1 to end]
+++ oracle: <full oracle path> [last N_back pages]

If a sliced comparison is byte-identical, write: `IDENTICAL — no diffs.`

### 5) Side-by-side format

Unchanged from B1.8.1 (80-char left column, pipe separator, `=== PAGE N ===` markers between pages).

### 6) Console output

After running, print a one-line summary per depo:
<stem>: front N pages, back M pages | front diff: K lines | back diff: J lines

Or for no-back-matter depos:
<stem>: front N pages, no back matter | front diff: K lines

### 7) Exit codes

Unchanged from B1.8.1:
- 0 — output files written successfully
- 1 — rendered or oracle file missing
- 2 — unexpected error

Still no pass/fail gate. Just slice and render.
