PROBLEM STATEMENT:
9-pair Stage A run on 2026-05-14 produced 631,209 diff events. Two
categorization bugs make the data unreadable:

BUG 1 — word_substitution is alignment garbage
- 139,912 events (22% of total volume)
- Sample: 'mail'->'Top', 'mail'->'Raise', 'pitre'->'hebert'
- Root cause: aligner pairs tokens that share no semantic slot
- These are NOT MB editing habits — they are alignment noise

BUG 2 — proper_noun_change mislabels Q/A speaker churn
- 9,540 events
- Top 3 patterns: 'Q'->'A' (103x), 'A'->'Q' (94x), 'I'->'Yes' (52x)
- Root cause: categorizer checks "is capitalized + len<=3" before
  checking if token is a Q/A speaker label
- Speaker label transitions are structural, not proper noun edits

FIX:

- Locate categorizer in src/aligner_differ/ (likely categorize.py
  or similar — read existing code first)

  as word_substitution, DROP it (do not emit to output)
- Add comment: "# word_substitution excluded — alignment noise, see
  A1 spec 2026-05-15"
- This is a deliberate dead category for v0.2. Do NOT keep the
  classifier branch and just filter downstream — drop it at source.

CHANGE 2 — Add q_a_speaker_change category, check BEFORE proper_noun.
- New category: q_a_speaker_change
- Trigger: raw_token in {'Q', 'A'} AND final_token in {'Q', 'A'}
  AND raw_token != final_token
- ALSO trigger: raw_token in {'Q', 'A'} AND final_token is None
- ALSO trigger: raw_token is None AND final_token in {'Q', 'A'}
- This check MUST run BEFORE proper_noun_change classification
- The 'I'->'Yes' pattern is a SEPARATE bug — leave for v0.3, do not
  fix in this spec (one bug per spec, clean rollback)
